EdgeOne 图片反代

透明反向代理 r.iirose.com 的图片资源，支持 CORS、智能缓存、断点续传

使用方式

部署后，将原本引用 r.iirose.com 的图片链接替换为你的 EdgeOne 加速域名即可， 路径和参数保持不变。

源站: https://r.iirose.com/i/25/2/5/13/3418-D2.gif
反代: https://你的域名.com/i/25/2/5/13/3418-D2.gif

功能特性

特性说明
透明代理原样转发路径和查询参数，无需修改现有 URL 结构
CORS 支持允许任意跨域请求，可安全用于前端 <img> 或 fetch
智能缓存图片缓存 1 年，其他静态资源缓存 1 天，动态内容不缓存
Range 支持支持断点续传，优化大图加载体验
错误处理统一返回 502 及错误信息，便于排查

API 参考

# 代理所有路径到 r.iirose.com
GET /{path} [?query]

# 预检请求
OPTIONS /{path}

# 响应头
cache-control: public, max-age=31536000, immutable
access-control-allow-origin: *
x-cache-status: HIT

部署说明

1. 在 EdgeOne 控制台 添加你的站点并完成接入

2. 进入 函数管理 → 新建函数，将 edge-functions/[[path]].js 中的代码粘贴部署

3. 设置触发规则，匹配路径为 /*（或你想要的路径前缀）

4. 在 EdgeOne 站点添加源站为 r.iirose.com 的加速域名， 或将函数关联到你的加速域名

提示: 如果只想代理特定路径（如 /file/*），可以在函数入口处增加路径判断逻辑。
